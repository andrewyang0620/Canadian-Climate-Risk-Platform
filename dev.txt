1. 最常用：启动本地开发服务
==================================================

推荐启动命令：

docker compose up -d postgres spark-master spark-worker airflow-webserver airflow-scheduler

说明：
- postgres: 项目主数据库，端口 5432
- spark-master: Spark UI，端口 8080
- spark-worker: Spark worker
- airflow-webserver: Airflow UI，端口 8081
- airflow-scheduler: Airflow 调度器

不要每次都运行 airflow-init。
airflow-init 只在第一次初始化，或数据库 volume 被删除后才需要。

==================================================
2. 第一次启动 Airflow / 重置后启动 Airflow
==================================================

如果是第一次跑 Airflow，先运行：

docker compose up airflow-init

然后再启动：

docker compose up -d airflow-webserver airflow-scheduler

Airflow 登录：
URL: http://localhost:8081
username: admin
password: admin

7. 停止服务
==================================================

停止并保留数据库数据：

docker compose down

危险：停止并删除数据库 volume：

docker compose down -v

注意：
docker compose down -v 会删除本地 Postgres 和 Airflow 数据库。
只有在你想完全重置环境时才用。

11. 常见问题
==================================================

问题：localhost:5432 打不开
原因：5432 是数据库端口，不是网页。
解决：用 DBeaver 或 psql 连接。

问题：Airflow打不开
检查：
docker compose ps
docker compose logs airflow-webserver

问题：Spark打不开
检查：
docker compose ps
docker compose logs spark-master

问题：Docker daemon error
解决：
打开 Docker Desktop，等 Engine running 后再运行命令。

问题：airflow-init exited with code 0
这不是错误。
它表示 Airflow 初始化完成并正常退出。
